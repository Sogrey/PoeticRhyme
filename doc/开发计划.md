# 书香诗韵 · 开发计划

## 一、项目概述

- **项目名称**：书香诗韵
- **项目类型**：国学启蒙学习平台（Vue 3 + WebAssembly）
- **核心功能**：古诗词、成语、经典名句的阅读、赏析、筛选与检索
- **目标用户**：小学生、儿童（6-12岁）、家长、教师
- **UI 框架**：shuimo-ui（水墨风 UI）
- **开发目录**：`e:\woekspace\study\PoeticRhyme`

## 二、技术架构

### 2.1 项目结构

本项目采用 **纯前端架构**，使用 WebAssembly 实现本地数据库操作：

```
PoeticRhyme/
├── frontend/              # 前端应用 (Vue 3 + Vite)
│   ├── src/
│   │   ├── components/   # 组件
│   │   ├── views/        # 页面
│   │   ├── stores/       # 状态管理
│   │   ├── services/     # API 服务层
│   │   ├── utils/        # 工具函数（含 SQLite 操作）
│   │   └── assets/       # 静态资源
│   ├── public/
│   │   ├── db/          # 预编译 SQLite 数据库
│   │   └── sql-wasm.wasm # SQLite WebAssembly 模块
│   └── package.json      # 前端依赖
├── scripts/              # 数据处理脚本（含 build-db.js）
└── package.json          # 根项目配置
```

## 六、数据存储方案

### 6.1 方案选择

**采用：预编译数据库 + sql.js 运行时**

**架构：**

```
编译阶段：JSON 数据 → build-db.js → poetry.db（预编译）
运行时：poetry.db → fetch → sql.js → 内存 SQLite → 前端组件
```

**理由：**

1. **完全本地**：所有数据存储在用户本地内存，无需网络连接
2. **性能优秀**：WebAssembly 处理数据库操作，速度快
3. **隐私安全**：数据不出浏览器，符合隐私要求
4. **开发简单**：无需后端服务，减少部署复杂度
5. **离线可用**：无网络环境下完全正常使用
6. **启动快速**：预编译数据库，无需运行时解析 JSON 和写入

### 6.2 数据流

```
编译阶段（scripts/build-db.js）：
JSON → better-sqlite3 → poetry.db

运行时（前端）：
poetry.db → fetch → sql.js → 内存数据库 → 前端组件
```
